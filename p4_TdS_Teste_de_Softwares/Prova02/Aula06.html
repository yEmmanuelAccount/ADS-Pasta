Testes de Unidade

Também conhecido como Teste Unitário, resulta de práticas de XP:

  • código coletivo
  • refatoração
  • integração contínua

Verifica a menor unidade possível do projeto de software (método, classe ou objeto).

Permite que sejam feitas alterações com mais segurança (menos medo de quebrar o código).

JUnit

É uma API de código aberto para a criação de testes unitários automáticos usando Java.

O objetivo principal é permitir a criação de testes, de forma fácil de escrever e executar.

Como? A partir de anotações e algumas declarações.

Testes basedos em Veredito

  • resultado esperado da execução de um único teste.

O teste é aprovado caso o teste atingiu o objetivo esperado e o software funcionou como esperado.

O teste é uma falha caso tenha atingido seu propósito, mas o software não teve o desempenho esperado.

ERRO: o caso de teste não atingiu seu propósito.

  • Ocorreu um evento inesperado durante o Caso de Teste
  • O Caso de Teste não pôde ser configurado corretamente.
