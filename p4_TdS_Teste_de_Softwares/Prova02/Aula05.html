Cenário de Teste

Cenário de teste é o caminho ou a situação a ser testada.

Exemplo 01: Transferência entre Contas

Passo a passo:

  1. Consultar o saldo da conta de origem
  2. Digitar os dados da conta de destino
  3. Confirmar se os dados estão corretos
  4. Transferir um valor da conta de origem para a conta de destino
  5. Digitar a senha
  6. Consultar novamente o saldo da conta de origem, verificando se o saldo inicial menos o valor a ser transferido é igual ao saldo atual

Casos de Teste:

  • CT_01: Preenchimento dos campos obrigatórios na tela de transferência
  • CT_02: Validação de CPF
  • CT_03: Conta Destino inválida
  • CT_04: transferência de valores negativos

Definição

Casos de Teste é um conjunto de dados de entradas, condições de execução do sistema e o resultado esperado/projetado para atingir um objetivo específico. Como, por exemplo, exercitar um caminho específico em um programa ou verificar a concordância do programa com o requisito.
Casos de teste é uma sequência de passos que devem ser executados no sistema, com os dados de entrada e saída esperada para cada passo especificados.

Casos de Teste

Servem como a base para projetar e implementar os testes reais.

É necessário desenvolver casos de teste para cada cenário de caso de uso.

Nem todo teste é complexo o bastante para justificar a sobrecarga de criação de um artefato de Caso de Teste que precise ser revisado e mantido.

Alguns dos Casos de Teste que você identificar inicialmente serão abandonados.

Características

  • Efetivo: testar o que planejou testar
  • Econômico: sem passos desnecessários
  • Reutilizável: pode ser repetido
  • Rastreável: é possível identificar o requisito a ser testado
  • Auto-explicativo: pode ser testado por qualquer testador

Exemplo 02: Casos de Teste

O programa deve retornar um valor.
  • Que tipo de valor deve retornar?
  • Está muito vago
  • nesse caso, valor pode ser qualquer coisa, inclusive algo não numérico

Atributos

Um caso de teste completo possui:

  • identificador do caso de teste
  • uma breve declaração de propósito
  • uma descrição de precondições
  • as entradas do caso de teste
  • os resultados esperados
  • uma descrição das pós-condições esperadas
  • um histórico de execuções

Como testar?

  • estabelecer precondições necessárias
  • observar resultados
  • comparar com os resultados esperados (oráculo)
  • garantir que as pós-condições esperadas existem para determinar se o teste passou

Especificações dos Casos de Teste

O título deve ser sucinto, simples e autoexplicativo, com informações para que o analista saiba a validação a que o teste se propõe.

Exemplos:

  • validar uploads de arquivos
  • validar cadastro de usuário com perfil de administrador
  • validar envio de ordem de compra

Objetivo

O objetivo do Caso de Teste é descrever o que será executado, fornecendo uma visão geral do teste que será realizado.

Exemplos:

  • verificar se realiza o upload de arquivo com as extensões permitidas
  • verificar se o cadastro é realizado após o preenchimento das informações
  • verificar se a ordem de compra é enviada informando o ativo, a quantidade e o preço

Pré-Condição

São condições necessárias para que o Caso de Teste consiga ser executado.

Exemplo:

  • Usuário cadastrado e autenticado no sistema
  • Ordem de compra enviada e executada
  • Usuário com perfil de administrador

Passos

Os passos são necessários para descrever todas as ações que o analista deve seguir durante a execução para chegar ao resultado esperado.

Deve iniciar com um verbo no infinitivo ou imperativo. Exemplos:

  • Acessar tela Negociação > Boleta
  • Clique no botão "Entrar"
  • Verificar se a edição foi salva no banco de dados
  • Preencha os campos do cadastro

Resultados Esperados

Descrever o comportamento esperado do sistema após executar os passos da fase anterior.

Deve ter os verbos no presente, evitando frases futuras usando "deve". As frases devem ser escritas como se a ação tivesse dado certo, descrevendo o comportamento esperado e sem deixar dúvidas.

Exemplos:

  • Sistema apresenta a tela de edição com os campos preenchidos
  • A ordem é enviada e executada com o preço informado
  • O cadastro é salvo no banco de dados

Importante!

Se um conjunto de testes não revelar nenhum erro, é possível que a consistência dele seja baixa ou que sua qualidade seja baixa.

Um bom Caso de Teste tem alta probabilidade de revelar um erro ainda desconhecido.

Quando elaboramos um conjunto de Casos de Teste, precisamos levar em consideração alguns critérios:

  • Consistência: é importante que o Caso de Teste seja elaborado em conformidade com as especificações do sistema testado.
  • Tamanho: tão grande quanto o seu orçamento permitir.
