Teste de Sistema

Verifica se o produto atende aos requisitos especificados e se o comportamento do programa está correto.

Ocorre após o Teste de Integração.

Valida se o sistema completo atende aos requisitos funcionais e não-funcionais e quão bem ele faz isso.

É chamdo também de Teste de Caixa Preta, já que é testado com tudo que é integrado e vai usar quando estiver em pleno funcionamento.

Deve ser desempenhado por profissional altamente qualificado do time de teste, em um ambiente fechado ao negócio.

A base de Teste de Sistema inclui todos os documentos todos os documentos ou informações descrevendo o objeto de teste no nível de sistema:

  • Requisitos de Sistema
  • Especificações
  • Análise de Risco, se presente
  • Manuais do Usuário; etc.

O Teste de Sistema testa o sistema como um todo o mais próximo possível do ambiente de produção.

Utilizamos os produtos de hardware e software necessários para o funcionamento do sistema como um todo.

Os testes de sistema não não devem usar drivers de teste nem stubs.

Por que testar após o Teste de Unidade e o de Integração dos Componentes?

Nos testes de nível menor, foi realizado o teste para verificar se as especificações técnicas, usando perspectiva do desenvolvedor do software.

O teste de sistema, por outro lado, olha o sistema sob a perspectiva do cliente e do futuro usuário.

Os testadores validam se os requisitos são completamente e apropriadamente implementados.

Muitas funções e características do sistema resultam da interação de todos os componentes do sistema que só podem ser acessados quando o sistema inteiro está implementado, podendo ser observado e testado.
