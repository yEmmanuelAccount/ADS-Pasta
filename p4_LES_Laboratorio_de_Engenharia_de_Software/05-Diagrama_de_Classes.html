Definição

“Um diagrama de classes descreve os tipos de objetos presentes no sistema e os vários tipos de relacionamentos estáticos existentes entre eles.”

    • (FOWLER)

O diagrama de classes representa os tipos (classes) de objetos do sistema que está sendo construído.

  • Propriedades
  • Funcionalidades
  • Relacionamentos com outras classes

Pode ser mapeado diretamente para uma linguagem orientada a objetos.

Classes

São representadas por retângulos contendo, respectivamente:

  • Nome (obrigatório)
  • Atributos (opcionais)
  • Operações (opcionais)

Exemplo de Código em Java:

        public class Cliente {
            private String nome;
            private String endereco;
            private Pedido pedido;

            // Constructor

            // GETs e SETs
        }

Atributos

Cada classe possui atributos e existem alguns elementos visuais que trazem informações sobre esses atributos:

Esses elementos são:

  • Visibilidade: "-", "~", "#", "+"
  • Nome: "nome", "endereco"
  • Tipo: "String", "Pedido"
  • Multiplicidade: "[0..*]"
  • Valor Padrão: "Fulano"
