Definição

“Um diagrama de classes descreve os tipos de objetos presentes no sistema e os vários tipos de relacionamentos estáticos existentes entre eles.”

    • (FOWLER)

O diagrama de classes representa os tipos (classes) de objetos do sistema que está sendo construído.

  • Propriedades
  • Funcionalidades
  • Relacionamentos com outras classes

Pode ser mapeado diretamente para uma linguagem orientada a objetos.

Classes

Atributos

Cada classe possui atributos e existem alguns elementos visuais que trazem informações sobre esses atributos:

Esses elementos são:

  • Visibilidade: "-", "~", "#", "+"
  • Nome: "nome", "endereco"
  • Tipo: "String", "Pedido"
  • Multiplicidade: "[0..*]"
  • Valor Padrão: "Fulano"

Visibilidade

Essa parte fica indicada graficamente antes do nome de cada atributo.

  • Privado (-): Somente a própria classe pode manipular o atributo.
  • Pacote (~): Qualquer classe dentro do mesmo pacote pode manipular o atributo.
  • Protegido (#): Qualquer subclasse pode manipular o atributo.
  • Público (+): Qualquer classe do sistema pode manipular o atributo.

Nome, Tipo e Valor Padrão

  • Nome: conforme o padrão da linguagem.
  • Tipo: primitivos ou classes.
  • Valor Padrão: valor de inicialização.

Exemplo: - nome : String "Fulano"
