Prototipação

O que é: é o desenvolvimento rápido de um sistema.

No passado, protótipos tinham a finalidade apenas de avaliar os requisitos.

Atualmente, os limites entre prototipação e o desenvolvimento normal do sistema, muitas vezes, são indefinidos e muitos sistemas são desenvolvidas usando uma linguagem evolucionária.

Não é simplesmente uma forma de validar a sua ideia, é uma parte integrante no processo de aprendizagem e inovação.

  • Arte de lidar com expectativas.

Motivação

Principal uso para ajudar clientes e desenvolvedores a entender os requisitos do sistema.

Por mais que o levantamento de requisitos traga uma infinidade de informações, é durante a interação real do usuário com o sistema que os detalhes realmente são percebidos.

Para projetar a experiência perfeita, a utilização de protótipos é a forma mais eficaz de simular essa interação.

O tempo gasto na prototipação é inversamente proporcional ao tempo necessário para a programação.

  • Quanto mais se dedicar na fase de prototipação, menos será o gasto de sua implementação.
  • Mais prototipação = menos gastos de implementação.

Vantagens

  • Baixa demanda de tempo para o desenvolvimento e, consequentemente, baixo custo.
  • Não requer conhecimentos avançados de software de edição gráfica.
  • Facilita a visualização do produto para o cliente desde a fase inicial.
  • Possibilita receber o feedback do cliente em tempo ágil.
  • Facilita o levantamento de requisitos e funcionalidades.
  • Possibilita estimar de forma mais precisa a complexidade e o tempode desenvolvimento.
  • Possibilita a realização de testes de interações.
  • Reduz os esforços de desenvolvimento.

Ciclo de Vida

  1. Coleta e Refinamento de requisitos
  2. Projeto Rápido
  3. Construção de Protótipo
  4. Avaliação do Protótipo por parte do Usuário
  5. Refinamento do Protótipo
  6. Engenharia do Produto

Classificações

Baixa Fidelidade

Também chamados de rascunhos ou sketches.

São concebidos ainda na fase inicial, durante a concepção do sistema.

Feito à mão utilizando lápis, borracha e papel.

Essa etapa é essencial para a definição do que é o produto e levantamento de requisitos.
