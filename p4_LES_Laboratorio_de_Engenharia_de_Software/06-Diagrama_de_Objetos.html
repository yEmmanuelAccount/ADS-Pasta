O que é?

Um Diagrama de Objetos é um diagrama estrutural de UML que mostra as instâncias das classes, usando notação semelhante à usada nos diagramas de classe. Quando representado visualmente, você verá muitas semelhanças ao diagrama de classes.

Representa os objetos e as ligações entre eles, assim como o diagrama de classes representa as classes e as associações entre elas.

Assim como os diagramas de classes, os diagramas de objetos modelam a visão lógica.

São úteis quando se quer descrever como os objetos do sistema interagem em um cenário particular.

Ideia

  • Retângulos representam objetos.
  • Linhas representam as ligações.

Objetos

A notação usada pelos Diagramas de Objetos deriva diretamente da notação dos Diagramas de Classes. A diferença é que apresenta os nomes dos elementos, que são as instâncias, sublinhados.

  • Modelo completo: nome_do_objeto: Classe
  • Modelo incompleto: nome_do_objeto
  • Modelo anônimo: :Classe

O Modelo Anônimo é usado para sabermos o nome da classe do objeto, mas o nome do objeto ainda não foi definido (ou não nos interessa).

O Modelo Incompleto é usado quando sabemos o nome do objeto, mas não se sabe ao certo a classe à qual ele pertence.

Os Retângulos que simbolizam os objetos podem comportar um segundo compartilhamento que contém o valor dos atributos.

O tipo dos atributos não é necessário porque já foi definido no diagrama de classes que engloba o dos objetos.
