Introdução a Linguagem Script para Web

Conceitos Fundamentais e Mercado

A Programação Web é o processo de criar sistemas e aplicativos que funcionam na internet. É um mercado de alta demanda com oportunidades globais.

Habilidades Necessárias

Para atuar na área, o profissional precisa equilibrar Hard Skills (conhecimento técnico de código) e Soft Skills (comunicação e resolução de problemas).

Tipos de Programação e Lógica

A lógica de programação define como o software se comporta e é dividida em dois grandes paradigmas:

Lógica Imperativa

Foca no como o programa executa, detalhando passos e sequências (ex: JavaScript).

Lógica Declarativa

Foca no que precisa ser feito, definindo regras e estados (ex: HTML e CSS).

Arquitetura Web e Internet

A internet funciona através de protocolos de comunicação como o TCP/IP. O TCP garante a transmissão dos dados enquanto o IP identifica os dispositivos.

Protocolos de Rede
  • DNS: Traduz nomes de domínio para endereços IP.
  • HTTP: Protocolo de transferência de dados da World Wide Web.

Divisões da Programação

  • Front-end: Interface e interação (HTML, CSS, JS).
  • Back-end: Lógica de negócio e servidor (Node.js, Python).
  • Fullstack: Atuação em ambas as frentes.
Modelos de Aplicação

Existem as Aplicações Tradicionais, que recarregam a página a cada ação, e as SPAs (Single Page Applications), que carregam dados dinamicamente via API para uma experiência mais fluida.

JavaScript e TypeScript

O ecossistema JavaScript evoluiu para incluir especificações e superconjuntos que tornam o desenvolvimento mais robusto.

JavaScript (JS) e ECMAScript

Linguagem de script interpretada e multiparadigma. O ECMAScript é o padrão que dita as regras da linguagem.

TypeScript

Criado pela Microsoft, adiciona tipagem estática ao JavaScript, facilitando a manutenção e detecção de erros precocemente.

Transpilação de Código

Ferramentas como Babel ou TSC convertem códigos modernos para versões compatíveis com navegadores antigos (ES5).

Boas Práticas de Código

Localização do Script

Recomenda-se colocar a chamada do JavaScript na última linha antes do fechamento do </body> para evitar bloqueios na renderização da DOM.

Organização de Arquivos

O uso de arquivos .js externos é preferível para melhorar a performance e facilitar a manutenção do projeto.
