Introdução a APIs e RESTful

HTTP Básico e Características

O HTTP (Hypertext Transfer Protocol) é a base da comunicação na Web, operando através do modelo Request-Response. Uma característica fundamental é ser Stateless (sem estado), onde cada requisição é independente.

Estrutura da Comunicação

Requisições (Requests)

Identificam recursos através de URIs e utilizam verbos para definir a ação, além de conterem cabeçalhos (Headers) com metadados.

Respostas (Responses)

Retornam um Status Code (como 200 OK ou 404 Not Found) e o corpo (BODY), que contém o recurso solicitado.

Web Services REST

O REST (Representational State Transfer) é um estilo arquitetural para sistemas distribuídos. APIs que seguem este modelo utilizam URIs para recursos e verbos HTTP para operações.

Formatos de Dados

Embora suporte múltiplos formatos, o JSON é o mais utilizado em APIs modernas por sua leveza e suporte nativo em JavaScript.

Restrições Fundamentais (Arquitetura RESTful)

Para que um serviço seja considerado RESTful, ele deve aderir a um conjunto de restrições arquiteturais:

Client-Server e Stateless

Separação clara entre as responsabilidades do cliente e do servidor, sem armazenamento de estado do cliente no servidor para garantir escalabilidade.

Cache e Layered System

Uso de cache para performance e arquitetura em camadas (como Load Balancers) para desacoplamento e segurança.

Uniform Interface

Uso de URIs coesas, mensagens autodescritivas e Hypermedia (HATEOAS), fornecendo links para recursos relacionados.

Mapeamento CRUD e Maturidade

Os verbos HTTP são mapeados diretamente para as operações de persistência de dados (CRUD).

Verbo HTTP Operação CRUD Idempotente?
GET Ler (Read) Sim
POST Criar (Create) Não
PUT Substituir (Update) Sim
PATCH Modificar (Update) Não
DELETE Excluir (Delete) Sim

Níveis de Maturidade e Segurança

Modelo de Maturidade de Richardson

Classifica a aderência ao REST, sendo o Nível 3 o ápice da maturidade ao utilizar controles de hipermídia.

Segurança em APIs

Devido à natureza Stateless, a autenticação evita sessões no servidor e utiliza métodos como OAuth2 ou JWT (JSON Web Tokens).
