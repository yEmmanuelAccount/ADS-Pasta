Arquitetura de Sistemas de Software

A arquitetura de sistemas é o modelo de mais alto nível das partes que compõem um software. Ela define a estrutura dos subsistemas, componentes e as propriedades visíveis de relacionamento entre eles.

Propósito da Arquitetura
  • Comunicação: Facilita o diálogo entre stakeholders (usuários e desenvolvedores).
  • Análise de Requisitos: Permite avaliar critérios críticos como desempenho e confiabilidade.
  • Reuso: Promove o aproveitamento de software em larga escala.

Visões Arquiteturais (Modelo 4 + 1)

Proposta por P. Krutchen, a documentação de um projeto deve ser representada graficamente através de diferentes perspectivas, tendo a Visão de Casos de Uso como o centro de tudo.

As 4 Visões Técnicas

Visão Lógica

Descreve a decomposição do sistema em abstrações e comportamentos, comumente representada por diagramas de classes e de sequência.

Visão de Processos

Foca na comunicação entre os processos do sistema, essencial para tratar concorrência.

Visão de Implementação

Descreve a organização dos módulos do sistema e seus componentes físicos.

Visão de Implantação (Física)

Define como a aplicação é instalada na rede, permitindo avaliar requisitos não-funcionais como disponibilidade.

Padrões de Arquitetura

Um padrão arquitetural é uma solução consolidada para problemas recorrentes no desenvolvimento de software.

Arquitetura em Camadas

Prática de dividir problemas complexos em partes menores e auto-contidas, diminuindo a dependência entre os módulos.

Camadas em Sistemas Corporativos
  1. Apresentação (Front-end): Interação com o usuário e tratamento de requisições HTTP.
  2. Domínio (Lógica de Negócio): Validações e regras que regem os dados.
  3. Fontes de Dados: Comunicação com bancos de dados e APIs externas.

Padrão MVC (Model-View-Controller)

O MVC divide a interface do usuário em três componentes distintos para facilitar a manutenção e os testes.

Componentes do MVC
  • Modelo (Model): Gerencia os dados e as regras de negócio. É independente da interface.
  • Visão (View): A camada de exibição (ex: páginas HTML).
  • Controlador (Controller): O intermediário que interpreta as entradas do usuário e atualiza o Modelo ou a Visão.
Boas Práticas: Separação de Preocupações

A separação entre a Visão e o Modelo é uma das práticas mais importantes da engenharia de software, permitindo que a lógica de negócio evolua sem depender de como ela é apresentada ao usuário.
