Programação Backend com NodeJS

Node.js e Backend

O Node.js é um ambiente de execução JavaScript no lado do servidor. Sendo uma plataforma open-source, permite que desenvolvedores utilizem a mesma linguagem do front-end para construir regras de negócio, conexões com bancos de dados e autenticação.

Arquitetura e Performance

A grande eficiência do Node.js vem da sua Arquitetura Event-loop, que opera de forma single-thread mas suporta I/O Non-blocking através da biblioteca libuv.

Responsabilidades do Backend
  • Gerenciamento de rotas e integrações.
  • Aplicação de regras de negócio.
  • Segurança e persistência de dados.

Gerenciamento de Pacotes (NPM)

O NPM (Node Package Manager) é a ferramenta padrão para instalar bibliotecas de terceiros, permitindo a expansão das funcionalidades do projeto de forma modular.

Comandos Fundamentais
  • npm init -y: Cria o arquivo package.json inicial.
  • npm install: Instala dependências (use -D para ferramentas de desenvolvimento).
  • npm remove: Remove um pacote do projeto.

Desenvolvimento de API REST

O backend funciona como um centralizador que processa requisições de múltiplos clientes e devolve respostas padronizadas.

Parâmetros de Requisição

Existem três formas principais de enviar informações para o servidor através das rotas:

Route Params

Parâmetros obrigatórios na URL, geralmente usados para identificar recursos únicos (Ex: /users/:id).

Query Params

Parâmetros opcionais usados para filtros ou paginação (Ex: /users?name=fabio).

Body Params

Dados enviados no corpo da requisição (geralmente em JSON), ideais para criação e edição (POST/PUT), evitando expor dados sensíveis na URL.

O Framework Express

O Express é o framework mais popular do ecossistema Node.js. Ele é "sem opinião" (unopinionated), oferecendo flexibilidade total para gerenciar rotas e configurações de porta do servidor.

Configuração e Ferramentas

Para iniciar um ambiente profissional de desenvolvimento, é necessário configurar o suporte ao TypeScript.

Setup com TypeScript
  1. Iniciar o projeto: npm init -y
  2. Instalar dependências: typescript, @types/node e tsx.
  3. Inicializar configuração: npx tsc --init.
Ferramentas Auxiliares

Para testar os endpoints da API, recomenda-se o uso do Insomnia ou Postman. Para a visualização de bancos de dados, o Beekeeper Studio ou o Prisma Studio são excelentes opções.
